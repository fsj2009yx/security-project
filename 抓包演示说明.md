# Kerberos认证体系抓包演示说明

## 一、抓包工具准备

1.  **工具名称**：Wireshark
2.  **下载地址**：https://www.wireshark.org/download.html
3.  **安装步骤**：按照官方安装向导完成安装

## 二、抓包操作步骤

### 2.1 启动抓包

1.  打开Wireshark软件
2.  选择本地网络接口（通常是以太网或WiFi接口）
3.  点击“开始捕获”按钮开始抓包

### 2.2 设置过滤条件

为了只捕获Kerberos认证体系相关的流量，设置以下过滤条件：

```
tcp port 5000 or tcp port 5001 or tcp port 5002 or tcp port 5003 or tcp port 5004 or tcp port 5005
```

### 2.3 触发认证流程

1.  打开浏览器，访问Client1客户端：http://localhost:5002
2.  点击“发起认证”按钮，触发认证请求
3.  点击“验证票据”按钮，验证票据有效性
4.  输入消息并点击“发送消息”按钮，测试加密通信
5.  点击“访问业务应用”按钮，访问V服务器业务应用

### 2.4 分析捕获的数据包

1.  在Wireshark中查看捕获的数据包
2.  重点关注以下内容：
    - 客户端与AS服务器之间的认证请求和响应
    - 客户端与V服务器之间的票据验证请求和响应
    - 加密消息的传输
    - 数字签名的传输

## 三、抓包演示要点

1.  **认证流程**：展示Client向AS服务器发送认证请求，AS服务器返回Ticket和会话密钥的过程
2.  **票据验证**：展示Client向V服务器发送Ticket和证书，V服务器验证票据有效性的过程
3.  **加密通信**：展示Client与V服务器之间的加密消息传输，包括明文、密文和签名
4.  **安全性验证**：通过抓包展示传输数据的加密性，确保敏感信息（如会话密钥、密码等）不被明文传输

## 四、注意事项

1.  抓包过程中确保网络环境稳定，避免其他无关流量干扰
2.  演示前确保所有服务器和客户端都已正常启动
3.  抓包完成后保存捕获的数据包，以便后续分析和演示
4.  演示时重点讲解Kerberos认证的核心流程和安全性保障机制
