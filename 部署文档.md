# Kerberos认证体系项目部署文档

## 一、运行环境要求

1.  **操作系统**：Windows 10/11
2.  **Python版本**：Python 3.7+
3.  **依赖库**：
    - Flask 2.0+
    - requests

## 二、安装依赖

1.  打开命令提示符（cmd）
2.  进入项目目录：
    ```
    cd C:\Users\OxO\Desktop\网安课程设计
    ```
3.  安装依赖：
    ```
    pip install flask requests
    ```

## 四、启动步骤

### 4.1 启动AS服务器

1.  打开命令提示符（cmd）
2.  进入项目目录：
    ```
    cd C:\Users\OxO\Desktop\网安课程设计
    ```
3.  启动AS服务器：
    ```
    python as_server/app.py
    ```
4.  服务器将在 http://localhost:5000 上运行

### 4.2 启动V服务器

1.  打开新的命令提示符（cmd）
2.  进入项目目录：
    ```
    cd C:\Users\OxO\Desktop\网安课程设计
    ```
3.  启动V服务器：
    ```
    python v_server/app.py
    ```
4.  服务器将在 http://localhost:5001 上运行

### 4.3 启动客户端

1.  打开新的命令提示符（cmd）
2.  进入项目目录：
    ```
    cd C:\Users\OxO\Desktop\网安课程设计
    ```
3.  启动Client1客户端：
    ```
    python client/app.py
    ```
4.  客户端将在 http://localhost:5002 上运行

5.  重复步骤1-3，启动其他客户端：
    - Client2：`python client2/app.py`（http://localhost:5003）
    - Client3：`python client3/app.py`（http://localhost:5004）
    - Client4：`python client4/app.py`（http://localhost:5005）

## 五、测试流程

1.  打开浏览器，访问任意客户端（如Client1：http://localhost:5002）
2.  点击“发起认证”按钮，测试认证功能
3.  点击“验证票据”按钮，测试票据验证功能
4.  输入消息并点击“发送消息”按钮，测试加密通信功能
5.  点击“访问业务应用”按钮，测试业务应用访问功能
6.  同时打开多个客户端，测试多Client并发登录功能

## 六、常见问题及解决方法

1.  **模块导入错误**：确保Python路径正确，或在代码中添加父目录到Python路径
2.  **端口被占用**：检查是否有其他程序占用了5000-5005端口，可修改代码中的端口号
3.  **依赖缺失**：确保已安装所有必要的依赖库
4.  **认证失败**：检查AS服务器是否正常运行，客户端ID是否正确

## 七、项目维护

1.  **代码更新**：修改代码后，需要重启相关服务器或客户端
2.  **日志查看**：服务器和客户端的运行日志会在命令提示符中显示
3.  **性能优化**：如需优化性能，可考虑使用生产级WSGI服务器（如gunicorn）

## 八、注意事项

1.  本项目为课程设计，仅用于学习和演示目的
2.  实际生产环境中，需要进一步加强安全性措施
3.  请勿在真实网络环境中使用本项目处理敏感信息
